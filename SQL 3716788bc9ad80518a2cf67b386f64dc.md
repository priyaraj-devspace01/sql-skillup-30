# SQL

## Definition of Data (SQL/DBMS)

> **Data is a collection of raw facts, figures, observations, or values that are stored in a database for future processing and retrieval.**
> 

### Definition of Information

> **Information is processed, organized, and meaningful data that helps users understand a situation or make decisions.**
> 

### Why database over spreadsheet or file manager

1. Faster Data Retrieval
    - Databases use SQL and indexes to find data quickly.
2. Better Security
    - Databases provide user roles, permissions, and access control.
3. Multi-User Support
    - Many users can access and update data simultaneously.
4. Data Integrity & Consistency
    - Constraints like Primary Key and Foreign Key prevent invalid data.
5. Scalability
    - Databases can handle millions of records efficiently, unlike spreadsheets.

In order to talk to this databases we need a language i.e, SQL(structured query language) .

### DBMS

The database itself is just a collection of organized data. It cannot independently manage user access, security, concurrent operations, backups, or data manipulation. Therefore, we use a **DBMS (Database Management System)**, which acts as an intermediary between users/applications and the database, managing the storage, retrieval, security, and manipulation of data.

### Database Server

The database is usually stored on a dedicated server, which is a powerful physical or cloud machine designed to handle large amounts of data, continuous operations, and multiple user requests reliably.

Unlike a typical personal computer, database servers provide high performance, better storage capacity, security, backup mechanisms, and 24/7 availability.

- **Server** → The machine (hardware/cloud VM) where everything runs.
- **DBMS** → Software installed on the server that manages databases.
- **Database** → The actual collection of stored data.

### 1. Relational Database (RDBMS)

```
- Data stored in tables.
- Uses SQL.
- Most widely used.
- Examples: MySQL, PostgreSQL, Oracle.
```

### 2. NoSQL Database

```
- Flexible schema.
- Handles large-scale and unstructured data.
- Examples: MongoDB, Cassandra, Redis.
```

### 3. Hierarchical Database

```
- Parent-child relationship.
- Tree-like structure.
- Example: IBM IMS.
```

### 4. Network Database

```
- Many-to-many relationships.
- Graph/network structure.
- Example: IDMS.
```

### 5. Object-Oriented Database

```
- Stores objects instead of tables.
- Suitable for OOP applications.
- Example: ObjectDB.
```

### Hierarchy

Server
│
└── DBMS
│
└── Database
│
└── Schema
│
└── Table
│
├── Columns (Fields/Attributes)
└── Rows (Records/Tuples)

Server      → Physical or cloud machine that hosts the DBMS.

DBMS        → Software that manages databases (e.g., MySQL, PostgreSQL).

Database    → Organized collection of related data.

Schema      → Logical blueprint that defines tables, relationships, and constraints.

Table       → Structure that stores data in rows and columns.

Column/Field→ A single attribute or property of the data (e.g., Name, Salary).

Row/Record  → A complete set of related data in a table representing one entity.

# SQL Data Types & SQL Commands Summary

## SQL Data Types

Data types define what kind of data a column can store.

### Numeric Data Types

| Data Type | Description |
| --- | --- |
| INT | Stores whole numbers |
| BIGINT | Stores very large whole numbers |
| DECIMAL(p,s) | Stores exact decimal values |
| FLOAT | Stores approximate decimal values |

### String Data Types

| Data Type | Description |
| --- | --- |
| CHAR(n) | Fixed-length string |
| VARCHAR(n) | Variable-length string |
| TEXT | Large text data |

### Date & Time Data Types

| Data Type | Description |
| --- | --- |
| DATE | Stores date (YYYY-MM-DD) |
| TIME | Stores time |
| DATETIME | Stores date and time |
| TIMESTAMP | Stores timestamp values |

### Boolean Data Type

| Data Type | Description |
| --- | --- |
| BOOLEAN | Stores TRUE or FALSE |

# Types of SQL Commands

SQL commands are mainly divided into 5 categories.

## 1. DDL (Data Definition Language)

Used to define and modify database structures.

| Command | Purpose |
| --- | --- |
| CREATE | Create database objects |
| ALTER | Modify existing objects |
| DROP | Delete objects permanently |
| TRUNCATE | Remove all records from a table |
| RENAME | Rename an object |

Example:

```sql
CREATE TABLE Employees(
    id INT,
    name VARCHAR(50)
);
```

---

## 2. DML (Data Manipulation Language)

Used to manipulate table data.

| Command | Purpose |
| --- | --- |
| INSERT | Add records |
| UPDATE | Modify records |
| DELETE | Remove records |

Example:

```sql
INSERT INTO Employees VALUES (1, 'Raj');
```

---

## 3. DQL (Data Query Language)

Used to retrieve data.

| Command | Purpose |
| --- | --- |
| SELECT | Fetch data from tables |

Example:

```sql
SELECT * FROM Employees;
```

---

## 4. DCL (Data Control Language)

Used to control user permissions.

| Command | Purpose |
| --- | --- |
| GRANT | Give permissions |
| REVOKE | Remove permissions |

Example:

```sql
GRANT SELECT ON Employees TO user1;
```

---

## 5. TCL (Transaction Control Language)

Used to manage transactions.

| Command | Purpose |
| --- | --- |
| COMMIT | Save changes permanently |
| ROLLBACK | Undo changes |
| SAVEPOINT | Create rollback points |

Example:

```sql
COMMIT;
```

---
